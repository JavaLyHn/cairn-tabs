# AI 整理预览 —— 清晰区分「有变更 / 无变更」

日期: 2026-07-17
状态: 已批准,待实现

## 背景

整理全部的预览(AIPlanDialog)会列出计划涉及的全部标签,但「真的移动了」与「本就在该组」
视觉上没区别,用户看不清这次到底改了什么。

## 决策(已确认)

方案 B:**只显变更 + 折叠无变更**,配顶部变更摘要。

## 判定「是否变更」

每个标签的当前组 `TabRecord.contextId`(`byId` 已可取)与目标比:
- 并入已有任务 `a`:`contextId !== a.taskId` → 移动;`=== a.taskId` → 无变更。
- 新建分组:组是新的 → 组内标签一律移动。
- 拿不准区:不算变更(单列)。

## 呈现(AIPlanDialog 重构)

1. **顶部变更摘要条**:`移动 N` / `新建 M 组` / `无变更 K` / `拿不准 L`,各计数为 0 则不显示;实时随编辑更新。
2. **新建分组**:整组皆移动,正常列;行加绿色左条(`border-l-2 border-accent bg-accent/5`);
   整理全部时行尾显示「原 X →」(复用 `aiPlan.tabSource` + 箭头,inbox 档无来源则不显示)。
3. **并入已有任务**:每组仅显示**移动进来**的标签(绿条 + 来源);本就在此组的收进可展开的
   「**N 个已在此组(无变更)**」折叠(展开后灰暗只读、无移除)。**移动数为 0 的并入组整组不显示**。
4. **拿不准 · 保持原位**:维持现状(琥珀 + 理由)。
5. **应用按钮**:移动数 `>0` → 「应用(N 处变更)」;否则「应用」。
6. **空态**:移动数与拿不准都为 0 → 正文显示「本次无需变更」,应用禁用。

## finalPlan(只应用真实变更)

- newGroups:保持(全为移动)。
- assign:每组 tabIds **过滤为仅移动项**(丢弃无变更标签),移动为空的组丢弃。
  → 应用时只移动真正变更的;无变更标签原位不动、不重排、不触发原组误删空。
- unclear:透传(点亮会话标记)。
- 既有编辑(改组名 / 移除组 / 移除移动标签)保留;移除某移动标签 = 取消该标签的移动(留原组)。

## 交互状态

- 折叠展开:`useState<Set<taskId>>`,点「N 个已在此组」标题切换。

## i18n(四语,新增)

`aiPlan.summary.moved`={移动 {n}}、`aiPlan.summary.newGroups`={新建 {n} 组}、
`aiPlan.summary.unchanged`={无变更 {n}}、`aiPlan.summary.unclear`={拿不准 {n}}、
`aiPlan.unchangedFold`={{n} 个已在此组(无变更)}、`aiPlan.applyN`={应用({n} 处变更)}、
`aiPlan.noChanges`={本次无需变更}。(来源沿用既有 `aiPlan.tabSource`)

## 测试

- 并入组:移动标签默认显示、无变更标签默认折叠(不显示)+ 折叠计数文案;展开后可见。
- 移动数为 0 的并入组不渲染。
- `onApply` 收到的 finalPlan:assign 只含移动标签(无变更被滤掉)。
- 摘要计数正确;空态「本次无需变更」+ 应用禁用。
- 既有测试(来源显示、favicon 兜底、拿不准区)保持通过。

## 取舍

- 无变更并入组不显示 / 折叠:焦点全在变更上,列表更短;需要时可展开核对。
- 判定基于生成方案时的 `contextId` 快照(与整个预览一致),编辑期间稳定。
