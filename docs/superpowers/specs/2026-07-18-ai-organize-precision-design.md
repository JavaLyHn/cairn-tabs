# AI 整理精度优化(减少归错组 / 重复建组)

日期:2026-07-18
状态:已确认(用户选定问题:①归错组/不相关塞一起 ②该并入已有任务却新建重复组)

## 背景

用户反馈 AI 整理「效果不好」。经确认,两大痛点都是**精度**问题:
1. **归错组**:把不相关的标签塞进同一组,或放进不合适的已有任务。
2. **重复建组**:明明有合适的已有任务,却又新建一个同类的组。

不是颗粒度问题、也不是太保守(unclear 机制保留)。

约束不变:F-13 隐私 —— 只发标签标题 + 主域名 + 任务名,不发完整 URL/query/页面内容。

## 根因

- 给 AI 的信号弱(标题+主域名),且提示词对「什么才算同一任务」缺乏明确门槛 → 模型按域名/类型/笼统主题硬凑。
- 「优先并入已有任务」只是一句软提示,没有硬优先级,已有任务的信号(3 条示例标题)也偏少 → 模型认不出匹配,倾向新建组。

## 方案(仅改 `core/ai/organize.ts`,解析逻辑与 JSON 结构不变)

### 1. 重写系统提示为「三步优先级」判断

对每个标签依次判断:
1. **能并入某个已有任务吗**?逐个对照 existingTasks 的 name/domains/samples,明显属于就 `assign`(最优先)。
2. 否则能和其它零散标签凑成「同一具体任务」的新组吗?能就 `newGroups`。
3. 都不行 → `unclear` + 简短理由(≤20 字)。

### 2. 抬高「同一任务」门槛(治①)

硬规则:「同一任务」= 明显服务于**同一个具体项目/目标**;仅仅同域名、同类型(都是搜索/文档/视频/社交)、
或主题笼统相关,**都不算**同一任务,不要硬凑。宁可少归、多留 unclear,也不要把不相关的塞进一组或塞进不合适的任务。

### 3. 硬化「先并入已有任务」(治②)

加规则:**禁止新建与某个已有任务主题重叠的分组 —— 那必须用 assign 并入,不要造重复组**。
`newGroups` 只用于确实没有对应已有任务的新主题。

### 4. 加一个紧凑 few-shot 示例

在系统提示里给一个 3 标签示意:一个并入已有任务、一个无关的留 unclear、一个并入已有任务。
小模型对这类示例特别敏感,对①②都直接见效。示例为固定文本、非用户数据,不涉隐私。

### 5. 增强已有任务信号

`summarizeTaskTabs` 的示例标题 **3 → 5** 条(域名仍 top5),让模型更容易认出「零散标签属于哪个已有任务」,减少重复建组。

### 保留

- 激进档(AI_ORGANIZE_ALL)仍可跨组移动/再平衡;默认档(AI_ORGANIZE_INBOX)仍保守。
- unclear「拿不准留原位 + 理由」机制不变。
- 提示词关键词兼容现有断言:conservative 含「保守」、aggressive 含「跨组」且不含「保守」、
  两档都含「unclear」「拿不准」、都不含「尽量」。

## 模型(旁注,不在本次代码范围)

默认 Haiku 聚类能力有限;设置里换 Sonnet/Opus 提升更明显。提示词改进与换模型可叠加。
